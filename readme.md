# 🚀 Proposta <a name="id01"></a>

Assim como Linus Torvalds disse "Falar é fácil, me mostre o código". Nós desenvolvedores nunca vamos aprender a programar e desenvolver software sem efetivamente codar, é como tentar aprender a andar de bicicleta lendo livros e vendo vídeos.

E foi assim que essa abordagem nasceu, um roadmap baseado em projetos. A ideia em seu princípio é simples, essa é uma demanda, tente encarar como um desafio técnico e completá-lo em até 7 dias.

Ao completar, não esqueça de publicar no linkedin e adicionar #handsOnRoadmap

## :trophy: O desafio <a name="id01.1"></a>

<br />

Imagine que um dia um arquiteto que está começando a sua empresa te procura, ele diz mais ou menos o seguinte.

"Olá, eu soube que você faz ótimos sites e vi aqui buscar seus serviços, afinal, já faz algum tempo que comecei minha empresa e o movimento é realmente parado, e acho que usar um bom site vai impulsionar meus negócios, além de aumentar a confiabilidade da minha empresa"

Ele expressa a vontade de tem uma bela página na web, e você será responsável por tirar isso do campo das ideias, seja criativo e busque usar o que sabe.

Ao final terá mais um ótimo projeto de portifólio.

<br />

## :dart: Os requisitos<a name="id01.2"></a>

### :dart: Requisitos funcionais <a name="id01.2.1"></a>

<br />

Seu site de arquitetura para o clente deve conter:

<br />

- O site precisa ser uma landing page
- deve conter uma das sessões mostrando trabalhos anteriores
- tem uma das sessões com depoimentos de clientes (usar nomes fake)
- ter uma sessão com entre em contato

### :dart: Requisitos não funcionais <a name="id01.2.2"></a>

<br />

É obrigatório a utilização de:

- HTML
- CSS
- scroll smooth
- menu de hambúrguer
- fazer deploy

<br />

### :pushpin: Requisitos não obrigatórios <a name="id01.2.3"></a>

<br />

Você será bem avaliado se usar:

<br />

- usar HTML semântico, como tags main, section...
- usar responsividade
- organizar e dividir bem os arquivos
- utilizar tema light e dark
